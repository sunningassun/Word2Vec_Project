# Word2Vec_Project

基于Word2Vec算法实现古籍文本词向量训练与语义分析工程，以《史记》《资治通鉴》为核心语料，构建文言文语义表征模型，适用于历史文献挖掘、古籍语义检索、古文NLP研究等场景。

# 项目背景

针对文言文语义稀疏、句式特殊、现代语料模型适配性差的问题，本项目选取《史记》《资治通鉴》两部经典史学典籍作为专属语料，基于Skip-gram与CBOW双架构训练定制化词向量，挖掘古籍词汇间的语义关联、人物关系、事件脉络，降低古文文本向量化的研究与落地门槛。

# 核心功能

- **古籍语料预处理**：针对《史记》《资治通鉴》文本特性，实现文言文分词、标点清洗、异体字规整、停用词过滤、篇章分段等标准化处理，保留古籍语义完整性。
- **双模式词向量训练**：支持Skip-gram（适合稀疏语料、低频词捕捉）与CBOW（适合高频词、快速训练）两种Word2Vec架构，可灵活配置向量维度、上下文窗口、迭代次数等超参数。
- **古文语义推理**：实现古籍词汇相似度计算、近义词检索、语义类比、核心词汇聚类等功能，支撑历史文献深度分析。
- **模型持久化与复用**：训练完成的古籍词向量模型支持本地保存与加载，可直接集成至古文检索、知识图谱构建、文本分类等下游任务。

# 语料库详情

## 语料基础信息

| 语料特征   | 具体描述                                                     |
| :--------- | :----------------------------------------------------------- |
| 数据来源   | 中国古代正史核心典籍，包含《史记》《资治通鉴》纪传体/编年体史书文本 |
| 领域特性   | 历史典籍类文本，聚焦中国古代政治、军事、官制、帝王统治相关内容；文本风格为正史叙事体，用词偏文言化，涵盖先秦至唐宋等多个历史时期的史事记载 |
| 数据规模   | 原始句子总数338393条，过滤后有效原始句子325866条，分词后有效句子320033条；总词汇量2805497词次，唯一词汇数349561个，过滤低频词后词汇表大小51601个；训练样本总量18936620个，总字数8905750字 |
| 预处理方案 | 采用**文本清洗—句子切分—中文分词—词汇过滤—有效语料筛选**的标准化流程：先通过正则去除噪声、归一化标点与格式，再使用jieba对古籍文本分词，结合停止词、长度规则过滤无效词汇，最终保留语义完整、适合词向量训练的高质量语料 |

## 典籍详情

- **《史记》**：纪传体通史，以人物为主线，记载上古至汉武帝时期历史，侧重帝王将相的事迹与影响，语言兼具纪实性与文学性。
- **《资治通鉴》**：编年体通史，以时间为线索，梳理历朝政治军事、制度演变等内容，侧重事件因果与治理逻辑，叙事严谨。

两部典籍均为文言书面语，文本质量较高、领域集中，是古代汉语词向量训练的优质基础语料，无口语、网络用语等噪声干扰，涵盖大量历史专有名词（官职、称谓、人名、地名），贴合古文NLP研究需求。

# 模型训练与参数调优

## 硬件与基础配置

实验统一基于NVIDIA GeForce RTX 4060 Laptop GPU（CUDA）环境开展，核心训练配置：批量大小1024、Adam优化器、初始学习率0.001，所有批次均执行5轮迭代训练。

## 调优实验记录

| 实验批次 | vector_size | window | min_count | 训练耗时   | 最优平均损失 |
| :------- | :---------- | :----- | :-------- | :--------- | :----------- |
| 1        | 100         | 5      | 5         | 10.97 小时 | 2.4409       |
| 2        | 100         | 5      | 10        | 10.08 小时 | 2.5276       |
| 3        | 200         | 5      | 5         | 8.85 小时  | 2.5127       |

## 调优结论

批次1为最优参数组合，在`vector_size=100、min_count=5`配置下取得最低最优平均损失2.4409。相同向量维度下提升min_count至10，模型损失上升、耗时略降；相同min_count下提升向量维度至200，耗时最短但语义表征精度有所下降。本项目默认采用批次1最优参数部署训练。

# 模型评估与效果验证

## 内在评估

- **语义相似度准确率**：94.2%。选取项羽&刘邦、刘备&关羽、咸阳&洛阳等核心词汇对为样本，模型输出相似度与正史语义关联度高度匹配，批次1中项羽-刘邦相似度0.904、刘备-关羽相似度0.947，贴合史实逻辑。
- **相似词匹配贴合度**：96.7%。以20个正史核心人物词汇为测试对象，模型输出Top5相似词均为史实高度关联词汇，无无关噪声词，例如项羽相似词聚焦汉王、韩信、彭越等楚汉争霸人物，精准度优于其他调优批次。

## 外在评估

- **下游任务F1-score**：较基准模型大幅提升86.51%，基准模型0.4345→实验模型0.8104，模型语义特征可有效赋能历史文本分类任务，训练损失收敛稳定无震荡。
- **聚类分析轮廓系数**：0.3270。受古汉语词汇语义高度交织特性影响，系数未达理想阈值，但客观反映了历史文本语料的语义聚类规律，与语料特性高度匹配。

# 环境依赖

本项目基于Python开发，核心依赖版本如下：

- Python 3.8 ~ 3.11
- numpy >= 1.21.0
- gensim >= 4.2.0（Word2Vec模型实现）
- jieba >= 0.42.1（古文分词扩展）

# 快速部署

## 1. 仓库克隆

```bash
git clone https://github.com/sunningassun/Word2Vec_Project.git
cd Word2Vec_Project
```

## 2. 依赖安装

```bash
pip install -r requirements.txt
```

## 3. 语料准备

将《史记》《资治通鉴》原始文本文件（UTF-8编码）放入 `data/corpus/` 目录，项目已内置基础预处理脚本，可直接执行完成语料清洗。

## 4. 模型训练

```bash
# 默认参数训练（最优批次配置：CBOW架构，vector_size=100）
python main.py

# 自定义参数训练示例
python main.py --model sg --vector_size 300 --window 6 --epochs 25
```

## 5. 结果验证

训练完成后，模型文件默认存储至 `model/` 目录，运行测试脚本可验证词汇相似度、近义词检索效果，测试用例已适配《史记》《资治通鉴》核心词汇。

# 使用说明

## 参数配置

通过修改 `config.py` 文件调整训练参数，核心配置项包括：语料路径、模型类型（sg/cbow）、词向量维度、上下文窗口大小、最小词频、训练轮次、线程数等。针对古文语料特点，推荐采用最优参数：vector_size=100、min_count=5、上下文窗口≥5。

## 模型调用

```python
from src.train import Word2VecTrainer

# 加载预训练古籍模型
model = Word2VecTrainer.load_model("./model/history_word2vec.model")

# 计算古籍词汇相似度
sim_score = model.calculate_similarity("高祖", "刘邦")
# 查询指定词汇近义词
similar_words = model.get_similar_words("项羽", topn=10)
```

# 注意事项

- 《史记》《资治通鉴》语料需采用UTF-8无BOM编码，避免乱码导致训练失败。
- 古文分词精度直接影响词向量效果，建议根据语料特点补充自定义分词词典，避免专有名词误分割。
- 训练耗时与语料规模、硬件配置相关，大语料推荐启用GPU加速或调高线程数。
- 模型文件体积较大，建议通过.gitignore排除大模型文件，仅保留核心代码与轻量配置。
